13

Product does not work

Device

The battery charges in less than 2.5 h.

With a fully discharged battery (LP451525), the battery is charged with a Keysight B2962 source meter unit in constant current mode (75 mA) until the charge voltage (4.2 V). Then charging mode switches to constant voltage mode. The charging process is terminated when the charge current drops below 31%.

The time elapsed between the start of the charging process, indicated in the description, of a fully discharged battery and the instant at which the charging current drops below 31% must be less than 2.5 h.

DEV-830 - Getting issue details... STATUS

14

Product does not work

Device

The battery charge level is obtained with a maximum error of 10% from the battery voltage using a lookup table taken from the battery calibration.

With a fully charged battery (LP451525), the discharge process is started at constant current (75 mA) while the battery voltage is monitored. Once fully discharged, a 15-point lookup table (LUT) is generated from the relationship between the charge drawn from the battery and its voltage. This LUT is used to estimate the battery charge as a function of the battery voltage. If the battery voltage falls between two points of the look up table, the charge level is obtained by interpolation.

With a fully charged battery (LP451525), the discharge process is started at constant current (75 mA) while the battery voltage is monitored. Once fully discharged, the battery level is calculated using the calibration LUT and the battery voltage, and compared to the actual charge level. This process is repeated with 5 different batteries. The error in all the estimations has to be less than 10%.

35

Malicious installation of a fraudulent user application update via over-the-air

Firmware

The device must be able to authenticate the update file received through the OTA update.

The update file must be signed with an asymmetric key that the device must verify to ensure that the file received is provided by Onalabs and not a third party.

  • Given a user application update file “A” signed by Onalabs with a asymmetric private key
    and a device with the asymmetric public key provided
    when the OTA update takes place and sends the update file “A”
    then the device installs the new user application version contained in update file “A” and boots the new user application.

  • Given a user application update file “B” non-signed by Onalabs
    and a device with the asymmetric public key provided
    when the OTA update takes place and sends the update file “B”
    then the device does not install the new user application version contained in update file “B” and boots the already installed user application.
