1. Purpose

To document baseline user journeys.

2. User journeys.

User Journey Patient Device/App: 

  1. A Patient signs up to the app.

  2. The app is paired and connected through Bluetooth Low Energy (BLE) to the device.

  3. As soon as there is pairing, the device initializes automatically and starts to send data to the APP.

  4. Data must be sent to the cloud in real time and stored and/or remotely displayed both graphically and numerically.

  5. If there is no connectivity with the Cloud, the device will store the data collected internally. As soon as there is connectivity, all the data will be automatically sent in bulk to the APP/Cloud.

  6. A data summary by date and time can be accessed through the APP.

  7. The Patient can log out.

  8. If the Patient is log out, the APP will stop collecting data to be sent to the Digital Platform repository.

User Journey Supervisors/Healthcare Professional Platform: 

  1. A Supervisor signs up through the web.

  2. The Supervisor can see all the Patients that he/she has in charge.

  3. The Supervisor can set thresholds on all the Health Features by Patient.

  4. The Supervisor can set the freqüency of data collection from the Physical Devices.

  5. The Supervisor selects the Patient he/she is interested in to visualize real time data. Also, a data summary by date and time of that Patient can be accessed.

  6. If the Patient have crossed any of the thresholds previously set, a visual alarm will be triggered.

  7. The Supervisor can assign one of the devices that he/she owns to a new Patient.

  8. The Supervisor can deassign one of the devices that he/she owns from a Patient.

  9. The Supervisor can log out.

User Journey Healthcare Provider/Admin Platform:

  1. An Admin signs up through the web.

  2. The Admin can establish which Patients and Supervisors can access the services.

  3. The Admin can assign devices to Supervisors under the restrictions of the license agreement.

  4. The Admin can deassign devices to Supervisors under the restrictions of the license agreement.

  5. The Admin can assign Patients and Supervisors to groups.

  6. The Admin can log out.
