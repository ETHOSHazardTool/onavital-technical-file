Purpose

Being able to obtain customer feedback or issues through different channels of easy access to the user ensuring quick and efficient replies for a great user experience. It must be a form that allows for scalability as well as categorization of the request type.

Digital Tools

The tool selected to serve to manage Customer Service is a Jira Service Management project which is a central place to collect, prioritize, assign, and track requests from customers. Besides, it provides integrated metrics interesting for Customer Service such as Time to Response or Satisfaction Polls. Furthermore, it is also linked to a knowledge base which the customer can access for troubleshooting, FAQsâ€¦

https://support.atlassian.com/jira-service-management-cloud/docs/what-is-a-service-project/

Customers can access every service project that you have given them permission to from help centers on the site or they can send requests via email, chat, or an embeddable widget. All requests from different channels are compiled in a board by category where the agents of the Service project can prioritize, respond and track from a centralized page.

Jira Service Request Example from Email Request

To test the procedure, an initial test was carried out by sending a mail to our specified mail: support.onavital@onalabs.com

  1. The request was created automatically in the board of tasks of the project:

  2. A confirmation mail is sent to the customer:

  3. When entering into the request, we can see the following settings:

    1. Summary field: Mail title

    2. Description: Mail body

    3. Contact is identified by the mail and signature of the body is also displayed.

    4. Request Type is Emailed Request by default but it can be changed after reception to better fit its nature.

    5. Similar requests can be found to link issues of the same nature.

    6. The default state is Waiting for Support.

  4. First response to customer to provide a first support or demand better explanation.

    1. The reply to customer can be handled directly as a comment on the issue to have all track of communications. To change state to the following one is recommended to respond to custommer by the State part where the same page opens up:

    2. The mail is received by the customer and the State of the Ticket is changed to Waiting for Customer.

    3. SLA of Time to Response automatically updated

      1. The user can press View Request on the mail received to also view the request in the Portal.

  5. Awaiting for Customer Response who can respond.

    1. The response is received in the board and the Status is changed to Waiting for Support.

    2. Another state can also be In progress which it is also set as a reply to Customer.

  6. The issue can be assigned to different persons in the team and even escalate the issue to another technical member.

  7. To close the issue, it can either be cancelled or resolved. It can be done by our agents and the user.

    1. The user must enter the View Request option in the Portal and Resolve or Cancel with a comment.

    2. The agent can cancel or resolve the issue warning the user if no response is received from customer.

    3. A final mail is sent to notify the closure of the ticket with a satisfaction quiz.
