1. Purpose

To describe user roles and role based access

2. User Roles

There are 3 distinct user tiers of the software

  • Tier 1 - Patient: Users who wears the physical device to have his health features monitored.

  • Tier 2 - Supervisor / Health Care Professional: Users who have different Patients under his surveillance and can access the data from the patients assigned to them.

  • Tier 3 - Admin / Health Care Provider: User who licenses the service and assign Physical Devices to Caretakers.

This table summarises the allowable tasks for each of the distinct user roles, based on the current baseline Functionality:

User Role

Task

Patient

Wear the ONA VITAL device accordingly to the user manual specifications.

Patient / Supervisor / Admin

Follow the ONA VITAL device instructions accordingly to the user manual specifications (pairing, battery charge, etc).

Patient / Supervisor

Ensure there is proper connectivity accordingly to the user manual specifications.

Supervisor

Assign ONAVITAL device to Patients under his surveillance.

Supervisor

Review data in real-time sorted by Patient.

Supervisor

Review historical data sorted by Patient and Date.

Admin

Enrol Patients and Supervisors to access the services under the restrictions of the license agreement.

Admin

Assign Patients to Supervisors so the former are under surveillance from the later.

Admin

Assign Supervisors to Groups.

Admin

Assign ONAVITAL devices to Health Care Professional / Caregiver.
