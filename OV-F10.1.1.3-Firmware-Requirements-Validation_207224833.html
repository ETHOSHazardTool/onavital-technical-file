Introduction

This document contains the results of the acceptance testing of the ONAVITAL’s firmware requirements from OV C3.1. Requirements Specification.

Test setup

ONAVITAL state is monitored through the SWV and its SWV ITM Data Console, and the Variables and Expressions views shown by STM32CubeIDE during debug sessions.

ONAVITAL external memory flash contents are read using the STM32CubeProgrammer.

The contents extracted from the external flash memory is converted to a legible format (CSV file) using the script located in Onalabs' GitHub repository.

To check the ONAVITAL GATT profile and its configuration, a third-party application has been used. The app, running in an Android device, is BLE Scanner from Bluepixel Technologies, version 3.25.

The mobile APP used to connect to the ONAVITAL runs in a smartphone plugged to a computer to monitor the output log.

To validate the requirements related to the connection/disconnection of the power cable to the ONAVITAL, an Analog Discovery 2 operated with the Waveforms application was used to apply a +5 V voltage to the input terminals of the ONAVITAL PCB when corresponding emulating the connection/disconnection of the power cable. Therefore, when said "connect the power cable" the Analog Discovery 2 output is enabled to generate a +5 V voltage at the ONAVITAL input terminals, and when said "disconnect the power cable"the Analog Discovery 2 output is disabled.

Description

Test 1 – 1

  • Configure a debug session (launch file) indicating as Initialization Commands, in the Startup tab:

    monitor flash mass_erase

This erases the microcontroller internal flash memory.

  • Start the debug session.

  • Place breakpoint on line 221 of the file Core/Src/app/active-objects/ao-ble/ao_ble_drivers.c.

  • Place breakpoint on line 315 of the file Core/Src/app/active-objects/ao-ble/ao_ble_drivers.c.

  • Configure in Expressions view the expressions BLE_HANDLES_ADDRESS and BLE_MAGIC_WORD_ADDRESS.

  • Run the code once.

Pass if

  • When stopped in breakpoint 221 for the first time, the values of the BLE_HANDLES_ADDRESS` members are equal to 0xFFFF.

  • When stopped in breakpoint 221 for the first time, the value pointed by BLE_MAGIC_WORD_ADDRESS is equal to 0xFFFFFFFFFFFFFFFF.

  • When stopped in breakpoint 315 for the first time, the values of the BLE_HANDLES_ADDRESS` members are neither 0xFFFF nor 0x0000.

  • When stopped in breakpoint 315 for the first time, the value pointed by BLE_MAGIC_WORD_ADDRESS is equal to 0xAAAAAAAAAAAAAAAA.

  • Restart the code execution (Run > Restart).

Pass if

  • When stopped in breakpoint 221 for the second time, the values of the BLE_HANDLES_ADDRESS` members are neither 0xFFFF nor 0x0000.

  • When stopped in breakpoint 221 for the second time, the value pointed by BLE_MAGIC_WORD_ADDRESS is equal to 0xAAAAAAAAAAAAAAAA.

Evidence

Result

Pass.
