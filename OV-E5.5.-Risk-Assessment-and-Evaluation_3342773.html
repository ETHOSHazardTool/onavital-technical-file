Patient/professional/ organisation

Poor early design.

Check the authenticity of the update file.

Sign the update files using an asymmetric key and, when an update is performed, the device checks the authenticity of the signature in order to proceed with the installation of the update.

An attacker who intercepts or gets hold of an update file can access the contents of the file and reverse engineer it to obtain the user application.
