1. Purpose

The purpose of this user interface evaluation plan is to:

  • Provide the documentation or the reference to the selected hazard-related use scenarios of ONAVITAL

  • Provide the summary of testing the safety of ONAVITAL

2. Scope

This user interface evaluation plan covers the references, revision history, scope, time schedule, formative evaluations and summative evaluation for ONAVITAL.

3. Formative evaluations

In the following section, all formative evaluation activities will be listed and shortly described. This includes a description of the aim of the evaluation as well as the applied method and the content being tested. If a usability test is applied, an additional description of the user groups attending the test, the test environment and the provided accompanying documentation and training will be given. Each usability test has a test protocol as a more detailed description. Additionally, each formative evaluation has an associated usability evaluation test report, that covers the results of the evaluation.

3.1 Formative Evaluation

Objective of evaluation

The objective of the formative evaluation will be to validate usability of release version 2.0.0 of the device which comprises the characteristics indicated in OV A1.6. Product History Record.

Conduction period

July 3rd 2023

Evaluated part of the user interface

An early stage version of ONALABS is evaluated. The early stage version comprises the following functions:

  • Cleaning.

  • Charging.

  • Sensor readings.

  • Historic data refreshes and updates.

Test method

For Formative Evaluation v2.0.0, a usability test (simulated use test + questionaire) will be used. The usability tests have a duration of a day. The test protocol can be seen in the document OV F10.1.2.4 Usability - Formative Evaluation.

Involved users and reference to the according user profile/user groups

The usability test will be conducted for users of the user group patients. Out of this user group, a total of 2 users will attend the test.

Test Environment and other conditions of use

The test will be conducted in a domestic environment. Participants were provided with a funtional prototype of ONAVITAL, a charger and disinfection material.

If applicable: Accompanying Documentation

All participants are provided with a version of the instructions for use.

Test analysis and documentation

All planned usability tests have been successfully executed. The results were analysed and evaluated in accordance with the applicable risk analysis, confirming that the user interface supports safe and effective use of the device under expected conditions of use. No usability-related risks requiring further corrective actions or user interface modifications were identified.

Therefore, no additional risk control measures or redesign activities have been deemed necessary. The usability evaluation confirmed compliance with the defined usability objectives.

The results of the usability testing are documented in the document OV F10.1.2.4.1 Formative Evaluation Report.

Test method

For the summative evaluation, a usability test (questionaire) with representative users will be used. This method analyzes whether the participants can manage tasks on the product without problems. They act under conditions of simulated use and perform tasks on the market equivalent device. Through observation, usage problems (e.g. incorrect operations, an uncertain handling, long processing times) can be identified.

Involved users and reference to the according user profile/user groups

The usability test will be conducted for users of the user group patients, supervisors and organizational admins. Out of this user group, a total of 16 users will attend the test. The participants characteristics are specified in OV C3.6.2 Usability - Use Specification.

Test environment and other conditions of use

The test will be conducted in a domestic environment. Participants were provided with a funtional prototype of ONAVITAL.

If applicable: Accompanying documentation

All participants are provided with the instructions for use and the corresponding questionaire.

Test analysis and documentation

All planned usability tests were successfully conducted in accordance with the defined protocol. The collected data included both qualitative and quantitative measures, obtained through direct observation, user feedback, task completion analysis, and error recording during simulated use scenarios with representative users under expected use conditions.

The data were captured using structured the questionaires forms. Particular attention was given to identifying any observed use errors, task failures, or patterns of confusion.

Each observed use error was analysed in correlation with the device's intended use and the associated risks identified in the risk management file. The analysis considered the severity, frequency, and detectability of the errors, and assessed whether they indicated a need for additional risk control measures.

No critical or unexpected use errors were identified during the tests. The results confirm that the user interface supports safe and effective use, and no further design modifications or corrective actions are required.

The outcomes of the usability evaluation are documented in OV F10.2.2.3 Usability - Summative Evaluation.
