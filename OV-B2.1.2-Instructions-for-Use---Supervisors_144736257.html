ONAVITAL Platform.

The Platform is intended for the use of the supervisor. The supervisor will be able to add users to its own group and assign devices to a user. Additionally, through the platform can access to the historic of a user.

Password Assignment and Reset

  • When a Supervisor is registered using an email account as credential, an email is sent to that user.

NOTE: If pressing the Verify Account button opens the browser and indicates that the invitation has expired, contact the Organization Administrator.

  • The Verify Account button on the Email sent you to a Password confirmation tab on the browser.

  • Once the user fill the Password confirmation, the user gets the notification of the Password confirmation.

  • If the user clicks again in the Verify Account button from the Email sent, they will be sent again to the Password Confirmation tab. However, if they want to change their password again, using this link, they will be denied.

  • To Reset the Password, the user must go to the Portal found here https://organization.app.cloud-onalabs.com/auth/login/main and click on the Password Reset link.

  • Once the supervisor click on the link, the user will be sent to the Reset Password tab, where they will introduce their email address. Once they click on the green button, an email will be sent to them.

  • The user will receive an Email to Reset their Password clicking on the Reset button.

  • That button will send the user to the Password Reset Portal where the user can set the new Password.

  • Once a new Password is set, the user will be communicated of the successful completion. And if the APP session is still running on the older credentials, it will be closed, so the new Password needs to be introduced.

Log-in

  • Enter to the Organizational Portal through this link: https://organization.app.cloud-onalabs.com/auth/login/main. Then enter the email and the password, finally click the button to Sign in.

  • If the credentials are correct, the supervisor will have access to the Organization Portal.

Add Users

  • Click on the Users tab.

  • Click on Add+, then a box will be displayed. Fill all the mandatory fields.

IMPORTANT: Even though is not mandatory, fill the fields Device Selection and Group.

  • Finally, click on Add and if all the fields were correctly filled the user will be added.

  • If any of the parameters were incorrectly filled or a parameter needs to be filled, an error will be displayed. Once the error is corrected, the user will be added.

Delete Users

  • Select de Users tab and identify the user you want to delete from the platform.

  • Press the three points displayed on the right side from the user. A menu is displayed, click on delete.

Update User information

  • Identify the user you want to update the information. Click on the user, a box on the right side will be displayed. Click on Expand.

  • A new screen is displayed. Select the Overview tab, click on Edit Details.

  • A new screen is displayed. Update the information from the user, then click on Save Changes. Now, the information from the user is updated.

Assign devices to Users

Organization administrators can assign free devices through the Devices tab.

  • Click on the Devices tab. In this tab the list of the devices assigned to the organization is displayed.

NOTE: Devices can't be added through this portal. This task can only be done by the Manufacturer Admin (ONALABS) through the Manufacturer Portal.

  •  Click on the device that does not have a user assigned. A box will be displayed on the right side.

  • Click on the EXPAND button, a new screen will be displayed.

  • Click on Edit Details. A box will be displayed with the fields that can be modified.

  • Select the user you will assign to the device, be sure that this user does not have a device assigned already.

  • Finally, press the button Save Changes to save the changes made.

Reassign a device to another user

If all devices have an assigned user, to reassign a device to another user, it is first necessary to verify that all Usage Sessions for the device are in the DONE state.

  • Select the device you want to reassign to another user. A box will be displayed on the right side, click EXPAND.

  • In the screen displayed, select Usage Sessions to Device tab. If the parameters Stop Reason and End Time are empty follow the steps of section Close Usage Session, otherwise follow the steps of the section No open Usage Sessions.

Close Usage Session

  • Identify the Usage Session that has the device that you want to reassign to another user. Once the session is identified, click on it. A right box will be displayed on the right side, click on EXPAND.

  • A new screen is displayed where the information regarding to the Usage Session selected is shown. Click on Edit Details.

  • A new screen is displayed. In the General tab fill the parameter End Time with the date and the hour, then select DONE for the parameter State.

  • Select the Summary tab and fill the Stop Reason with Done, and select Completion for the Stop Reason Code. Click on Save Changes.

  • Now, you will see the Usage Session with the Stop Reason and End Time updated. Now you can reassign the device to another user, following the steps from the section Assign devices to Users.

No open Usage Sessions

  • Identify the Usage Session that has the device that you want to reassign to another user. If the parameters Stop Reason and End Time are filled, then you can follow the steps fom the section Assign devices to Users.

View data in Real-Time

Supervisor can see the last data received from the user using the Supervisor Station tab.

  • Click on the Supervisor Station tab. In this tab all the users are displayed in a card, with the latest values of each signal vital.

View history

There are two ways to see the history data of a selected user.

First form

The firt form and the easiest one, is through the Supervisor Station tab.

  • Click on the Supervisor Station tab, and identified the user for whom you want to see the history, and click on the Monitor button

  • Then you will be redirected to a new screen, which is the Measurements tab from the selected user and you are in the History tab.

  • Select the start date of the history, first set the date and then the hour. Then click Apply.

  • Finally, the history data is displayed.

Second form

  • Click on the User tab and select the user you want to see the history. A box in the righ side will be displayed.

  • Click on EXPAND, and you will be redirected to a new screen with all the information from the user.

  • Click on the Measurements tab, which lead you to the screen where the graphs are shown. You are in the History tab.

  • Select the start date of the history, first set the date and then the hour. Then click Apply.

  • Select the view range, this means the time range that you want to display in the graph starting from the start date. Finally, the history data is displayed.
