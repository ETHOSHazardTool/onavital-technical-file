1. Purpose

To detail final product testing protocol.

2. Protocol

The test described below is carried out with 1 % of the manufactured ONAVITAL devices. This test is the prior step to release a manufacturing batch and it must be conducted as follows:

  1. Pick one packaging containing an ONAVITAL device.

  2. Visually inspect the package to ensure that it is in perfect condition.

  3. Open this packaging and extract the device.

  4. Visually inspect the device to ensure that it is in perfect condition.

  5. Open the plastic case using the housing opening tool.

  6. Characterize the electronic board as it is indicated below:

Power supply: First, use a multimeter to measure the response of each power supply circuit. Activate the system using the charging cable and upload on the microcontroller a testing firmware that enables the power supply system. Then, measure the voltage provided by each power supply circuit as it is indicated on the following table:

Circuit

Expected response

Test points

5.0 V Boost converter

5.0 V ± 5 %

TP3 (+5V) and TP4 (GND)

3.0 V LDO

3.0 V ± 5 %

TP2 (+3V) and TP4 (GND)

1.8 V LDO

1.8 V ± 5 %

TP1 (+1V8) and TP6 (GND)

Vbat

1.8 V ± 5 %

TP12 (Vbat) and TP6 (GND)

Vdd PPG

5.0 V ± 5 %

TP5 (VDD_PPG) and TP6 (GND)

If the recorded response is within the expected range, the result of this test is PASS. When this test is passed, the next step of the testing process is carried out. Otherwise, if the test is not passed, a NC form is raised and/or CAPA if necessary.

Control unit: Next, upload on the microcontroller a testing firmware that blinks the RGB LED. This LED must shine must shine in green, blue, and red 1 second each as it is shown in the following image:

If the response shines as it is expected, the result of this test is PASS. Otherwise, if the test is not passed, a NC form is raised and/or CAPA if necessary.

Communication module: Then, upload on the microcontroller a testing firmware that configures the BLE system. Use the application BLE Scanner to find the device and establish a remote connection with this system as it is indicated on the following image:

If the module is programmed successfully and the connection is established, the result of this test is PASS. Otherwise, if the test is not passed, a NC form is raised and/or CAPA if necessary.

Sensors: Once the communication module is validated, upload on the microcontroller a testing firmware that reads the ID of each sensor.

Sensor

Expected ID

IMU

0x6B

PPG1

0x15

PPG2

0x15

Temperature sensor

0x117

If the recorded IDs match with the expected ones, the result of this test is PASS. When this test is passed, the next step of the testing process is carried out. Otherwise, if the test is not passed, a NC form is raised and/or CAPA if necessary.

External memory: Eventually, upload on the microcontroller a testing firmware that reads the ID of the external flash memory. If the recorded ID is 0x4018, the result of this test is PASS. When this test is passed, the next step of the testing process is carried out. Otherwise, if the test is not passed, a NC form is raised and/or CAPA if necessary.

  1. Report the result of the aforementioned test on OV F10.100 Product Release - Validation & Verification for the release approval.
